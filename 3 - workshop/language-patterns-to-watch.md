# Language Patterns to Watch

Common phrasing patterns that create avoidable legal risk, with a quick fix for each. Pulled out on its own as a fast self-check before submitting a draft.

| Instead of... | What trips people up about it | Easy fix |
|---|---|---|
| Advertising something as "free" without disclosing what happens after a trial period, or what's required to get it | Many states and the FTC treat this as a disclosure issue. If a "free" offer auto-converts to paid, or requires a credit card up front, that condition needs to be clear, not buried in fine print. | State the condition next to the word "free." "Free for 14 days, then $9/month unless canceled." |
| Calling a feature "unlimited" when there's a practical or technical cap (storage, usage, support response time) | If a typical user could realistically hit a real limit, "unlimited" becomes a claim that can be challenged as misleading once someone hits that ceiling. | Either remove the cap, or describe what's actually offered. "Generous storage for most users" or state the real number. |
| Using a customer testimonial or review without disclosing whether it was incentivized, or presenting an unusual result as if it's typical | FTC endorsement rules require disclosing material connections (discounts, free product, payment) behind a testimonial. Showing a standout result without context can imply an outcome most customers don't actually get. | Disclose any incentive given for the testimonial, and add a brief qualifier if the result shown isn't typical. "Results vary; this reflects one customer's experience." |



## Reminder to  drafters 
This list reflects recurring patterns, not a complete list. If something doesn't fit a pattern above but still feels risky, that instinct is usually right — flag it.

# Runbook Maintenance

A runbook that goes stale is worse than no runbook — people lose trust in it and stop using it, which sends everything back to ad hoc requests.

## Cadence

- **Quarterly:** Review the request log for patterns — which non-Clearway requests kept coming up? Those become new Clearway entries.
- **Quarterly:** Spot-check existing Clearway entries against current product/legal reality. Anything outdated gets flagged or pulled.
- **As-needed:** Any time a Full Review results in a reusable answer, add it to Clearway immediately rather than waiting for the quarterly cycle.

## Version history

Keep a simple changelog at the bottom of each major doc, or use version control commit history if this lives in an actual repo. At minimum, track: what changed, when, and why.

## Signs the runbook needs an update

- The same non-Clearway question keeps coming up
- People are routing around the runbook (going straight to a person instead of checking docs first) — usually a sign the docs are out of date or hard to find
- A Clearway entry was used and turned out to be wrong
